# agy 定时自动对话工具 (agy-scheduler)

这是一个基于 [Bun](https://bun.sh/) 和 TypeScript 编写的轻量级定时工具。它能够按照您设定的时间规则，在每天指定的时间段内，每隔固定分钟数自动调用本地的 `agy` 命令行（Antigravity CLI），发送 “你好” 进行一次非交互式的对话。

## 功能特性

- **时间规则高度可配置**：支持配置每日开始时间、截止时间以及调用频率间隔（单位：分钟）。
- **执行命令可配**：默认调用系统中的 `agy --prompt "你好"`，也可通过配置文件修改具体的可执行文件路径与运行参数。
- **高精度且鲁棒的调度算法**：区别于容易产生累计漂移的 `setInterval`，本工具通过确定性的时间差计算进行 `setTimeout` 休眠，即使系统睡眠或休眠唤醒后，也能自动恢复并定位到下一个正确的调度时刻。
- **配置热重载 (Hot-Reload)**：守护进程在每次到达调度时刻时，都会重新读取并加载 `config.json`，修改配置无需重启守护进程。
- **完整的自动化测试**：基于 `bun test` 编写了高密度的调度器单元测试与执行器接口测试，确保核心逻辑 100% 绿灯。

## 快速开始

### 1. 安装依赖

本项目使用 Bun 作为运行环境。如果您尚未安装 Bun，请先前往 [Bun 官网](https://bun.sh/) 进行安装。

在项目根目录下执行以下命令以安装依赖（如 TypeScript 类型定义）：

```bash
bun install
```

### 2. 配置文件说明

项目根目录下包含 `config.json` 配置文件。您可以根据实际需要调整以下字段：

```json
{
  "scheduler": {
    "startTime": "08:00",       // 每天定时任务启动时间 (HH:MM 格式)
    "endTime": "23:30",         // 每天定时任务截止时间 (HH:MM 格式)
    "intervalMinutes": 30       // 调度周期，单位：分钟 (如 30 代表每半小时执行一次)
  },
  "command": {
    "executable": "agy",        // 本地 agy 命令的可执行文件名称或绝对路径
    "args": [                   // 传递给命令的参数列表
      "--prompt",
      "你好"
    ]
  }
}
```

### 3. 运行指南

#### 守护进程模式 (后台定时运行)

默认以守护进程模式运行。启动后，程序会计算下一个执行时刻，显示倒计时并挂起：

```bash
bun start
```

或显式指定参数：

```bash
bun run src/cli/index.ts --daemon
```

*如果想指定其他配置文件路径，可以使用 `--config` 参数：*
```bash
bun run src/cli/index.ts --daemon --config ./my-config.json
```

#### 单次测试模式 (立即运行一次)

如果您想立刻调用一次 `agy` 命令，验证命令是否正常，可以使用 `--once` 选项。该模式不会进入定时挂起循环，执行完毕后直接退出：

```bash
bun run src/cli/index.ts --once
```

## 自动化测试

项目遵循测试驱动开发（TDD）规范。您可以通过以下命令运行全部单元测试：

```bash
bun test
```

测试用例涵盖了：
- **调度时刻边界情况**：如跨天切换（23:30 跨越至次日 08:00）、正好处于启动点（08:00）的次轮计算、自定义频率等。
- **命令执行适配器**：测试子进程的成功调用、stdout/stderr 的流式捕获及进程退出码处理。
